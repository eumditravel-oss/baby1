## 0. 제품 한 줄 정의와 리브랜딩

**"엄마 아빠 목소리로, 언제 어디서든 아이에게 동화를 읽어주는 앱."**
수면은 여러 사용 장면 중 하나일 뿐이다. 등원길 차 안, 밥 먹을 때, 놀이 시간, 그리고 잘 때 — 모든 순간이 대상이다.

### 0.1 네이밍 변경 (수면 단어 전면 제거)
| 기존 | 변경 | 이유 |
|---|---|---|
| 앱 이름 (수면 컨셉) | **후보 ① 말랑동화 ② 이야기주머니 ③ 포포의 서재** | 발음 쉬움, 아이·부모 모두에게 각인, "언제든" 컨셉. 최종 확정 전 상표검색(KIPRIS)·스토어 중복 검색 필수 |
| 수면도서관 탭 | **동화 서재** | 콘텐츠 보관소라는 본질만 남김 |
| 꿈나라라디오 탭 | **이야기 라디오** | 라디오 메타포는 유지(연속재생 컨셉이 좋음), 꿈/수면 뉘앙스만 제거 |
| (신규) 핵심 기능 탭 | **우리집 목소리** | 엄마·아빠 목소리 기능을 탭 레벨로 승격 — 이 앱의 존재 이유이자 구독 상품 그 자체 |
| 마법의 책 | 유지 | 이름은 좋음. 수익 구조만 변경(2.4장) |

이하 문서에서 앱 가칭은 "이야기 서재"로 통일한다. 최종 이름 확정 시 전역 치환.

### 0.2 기존 50편 콘텐츠와 리브랜딩의 충돌 처리
1~50편은 전부 "주인공이 잠드는 결말"이다. 이건 버릴 필요 없다 — 아이들 그림책도 잠드는 결말이 많지만 낮에도 읽는다. 처리 방침:
- 1~50편은 **"포근한 이야기" 팩**(category: cozy)으로 분류해 그대로 서비스
- 다음 배치 51~100편은 6.2장의 신규 프롬프트로 **"신나는 모험" / "반짝 일상" 팩**을 생성 (잠들지 않는 결말, 낮 청취용)
- 홈 화면은 시간대 감지로 팩을 추천: 오후엔 모험 팩, 저녁 8시 이후엔 포근 팩 상단 노출

## 1. 제품 구조 (탭 5개, 출시 스코프)

### 1.1 동화 서재 (홈)
- stories_v2.json 기반 카드 그리드. 카드 = Flow 생성 표지 + 제목 + 재생시간(duration_est_sec)
- 카드 탭 → 플레이어: 목소리 선택(포근한 선생님 / 신나는 선생님 / 우리집 목소리), 배속(0.8/1.0), 15초 이동, 잠금화면 미디어 컨트롤(필수 — 부모는 화면 끄고 튼다)
- 리스트 3번째 슬롯에 네이티브 광고 1개(2.3장)

### 1.2 이야기 라디오
- **순차재생**: 1편부터 이어듣기, 마지막 청취 위치 저장
- **셔플재생**: 팩 단위 셔플(포근 팩만 / 전체). 직전 5편 재생 제외 규칙으로 체감 랜덤 개선
- **연속재생 타이머**: 1편 / 3편 / 30분 / 무제한. 저녁 자동 페이드아웃 옵션(볼륨 서서히 감소 후 정지) — "수면"이라 부르지 않아도 수면 수요를 흡수하는 장치
- 트랙 전환 사이 1.5초 무음 + 다음 화 제목을 나레이터 목소리로 예고("다음 이야기는, 느릿느릿 달팽이 두리…") — 사전 생성해 캐시, 라디오 감성의 핵심
- 구현 요건: 백그라운드 오디오 서비스, 오디오 포커스 처리, 블루투스/카플레이 대응

### 1.3 우리집 목소리 (구독 핵심)
파이프라인: **녹음 → 클로닝 → 동화별 생성 → 보관함**
1. 녹음: 앱 내 가이드 스크립트(6.5장) 낭독 2~3분. 소음 측정해 기준 미달 시 재녹음 유도. 화면에 "본인 목소리 사용 동의" 체크(ElevenLabs 약관상 필수, 법적으로도 안전판)
2. 클로닝: ElevenLabs IVC(Instant Voice Cloning) API. 유저당 보이스 슬롯 2개(엄마+아빠)
3. 생성: **구독 시 50편 일괄 생성 금지.** 유저가 재생 버튼을 누르는 시점에 온디맨드 생성 → 서버/스토리지에 영구 캐시. 같은 편 재생성 없음 (원가 통제의 핵심, 2.2장)
4. 보관함: 생성 완료된 편이 쌓이는 화면. "뭉게 이야기 · 엄마 목소리 · 2026.7.3" 형태 — 자산이 쌓이는 게 눈에 보여야 락인이 작동한다
5. 다운로드 금지, 스트리밍 전용 + 로컬 캐시 암호화. (mp3로 빼낼 수 있으면 구독 잠금 전략 전체가 무력화된다)
6. **변환 범위(v2.1 명시):** 목소리 등록 완료 시 **동화 서재 전 편(현재 50편 + 이후 추가되는 모든 팩)과 마법의 책 생성분**을 우리집 목소리로 청취할 수 있다. 이야기 라디오도 우리집 목소리 모드를 지원한다 — 미생성 편은 재생 직전 온디맨드 생성하고, 라디오 연속재생 중에는 다음 트랙 1편을 프리페치해 대기시간을 숨긴다. 편당 생성은 1회뿐(영구 캐시)이므로 원가 상한은 2.2장 헤비 케이스와 동일하다

### 1.4 마법의 책
- 아이 이름·좋아하는 것·오늘 있었던 일을 입력하면 Claude API가 즉석 맞춤 동화 생성 → 선택한 목소리로 TTS → 보관함 저장
- **구독 전용, 월 10회(서버 강제 카운트, 미사용분 이월 없음).** 비구독자는 가입 시 1회 무료 체험 — 단 체험 생성은 **나레이터 목소리로만**(원가 최소화 + "우리집 목소리로 들으려면 구독" 전환 장치). 광고 기반으로 열지 않는 이유는 2.4장
- **산출물 정책(v2.1):** 맞춤 동화 오디오는 우리집 목소리와 **동일한 과금·잠금 정책**을 적용한다 — 스트리밍 전용·암호화 캐시·다운로드 금지, 해지 시 재생 중지·보관함 보존·재구독 즉시 복구
- 같은 맞춤 동화 재청취는 캐시 재생으로 횟수 미차감. 내용을 바꿔 새로 생성하면 1회 차감. 월 10회 소진 시 회수권 IAP(2.6장)
- 생성 프롬프트는 6.2장 프롬프트를 서버 측에 내장, 금칙어 필터(실존인물·폭력·공포) 통과 후 TTS

### 1.5 설정
- 구독 관리(스토어 결제 연동), 목소리 관리(재녹음/삭제), 광고제거 안내, 이용약관·개인정보처리방침·**음성 데이터 처리 고지**(클로닝 데이터 보관·삭제권 명시)

## 2. 수익 모델 — 숫자와 판정

### 2.2 원가 구조 (가격 설계의 근거)
ElevenLabs API 종량 기준 <cite index="5-1">TTS는 Multilingual v2/v3가 1,000자당 $0.10, Flash/Turbo가 $0.05</cite>. 동화 1편 평균 약 450자이므로:

| 항목 | 원가(편당) | 비고 |
|---|---|---|
| 부모 목소리 TTS (v2) | 약 ₩65 | 450자 × $0.10/1k, 환율 1,400원 가정 |
| 부모 목소리 TTS (Flash v2.5) | 약 ₩33 | 품질 A/B 후 결정. IVC 지원 |
| 마법의 책 1회 (Claude 생성+TTS) | 약 ₩100~150 | 스토리 생성 API 비용 포함 추정 |
| 헤비유저 최악 케이스 | 약 ₩3,000~6,500 (1회성) | 목소리 2개 × 50편 전부 생성. 캐시 후 재발생 없음 |

→ 결론: **구독 첫 달에 헤비유저도 원가 회수, 2개월차부터는 서버비 외 원가 0에 수렴.** "이미 생성된 음성은 TTS 비용이 안 든다"는 가설은 수치로 성립한다. 단, 반드시 온디맨드+영구캐시 구조여야 한다(일괄 생성하면 이탈 유저에게 원가만 태운다).
> 요금·환율은 변동 가능. 출시 전 ElevenLabs 요금 페이지 재확인 항목으로 7장 게이트에 포함.

### 2.1 구독 가격 — 가격저항 분석과 권고안
한국 시장 심리 기준선: 광고형 OTT 5,500원, 음악 스트리밍 7,000~9,000원, 독서 구독 9,900원. 부모의 "아이를 위한 지출"은 자기소비보다 저항이 낮지만, **1만원이 명확한 심리적 벽**이고, "커피 한두 전" 프레임이 통하는 구간은 5,000~8,000원이다. 이 앱은 단일 킬러기능(내 목소리) 앱이므로 콘텐츠 구독(밀리 9,900)보다 반 발 아래가 맞다.

**권고안:**
- **월 6,900원 / 연 49,000원(월 4,083원 상당, "58% 아낌" 표기)** — 연간 유도가 곧 락인
- 7일 무료체험: 목소리 1개 클로닝 + 동화 3편 생성 허용. 체험 중 만든 "우리 아이가 들은 엄마 목소리 동화"가 체험 종료와 함께 잠기는 구조 자체가 최강의 전환 장치다
- 출시 첫 달 프로모 4,900원은 선택지로 보류(초기 리뷰 확보가 급할 때만). 가격을 낮게 시작하면 올리기 어렵다
- 구독 포함: 우리집 목소리 무제한 청취(**동화 서재 전 편 + 추가 팩 + 마법의 책 생성분 변환**) + 마법의 책 월 10회 + 전 스토리 팩 접근 + 전 화면 광고 제거

### 2.5 음성 잠금(락아웃) 전략 — 솔직한 판정
전략 자체는 유효하다. 클라우드 스토리지·SaaS의 표준 모델("해지하면 접근 불가, 데이터는 보존")이고 스토어 정책상으로도 구독 종료 후 기능 잠금은 정상 패턴이다. 다만 그대로 시행하면 터질 지뢰 3개와 완화책:
1. **별점 테러·환불 분쟁**: "내 목소리인데 왜 못 듣냐"는 리뷰가 반드시 나온다. → 구독 화면·해지 화면·약관 3곳에 "해지 시 재생이 중지되며, 보관함은 삭제되지 않고 재구독 즉시 복구됩니다"를 명시. '삭제가 아니라 보관'이라는 프레임이 방어선이다. 해지 후에도 보관함 목록·표지는 보이게 하라(자산이 남아있음을 시각화 = 재구독 유인 + 분쟁 방어 동시 달성)
2. **기술적 구멍**: 다운로드 가능하거나 캐시가 평문이면 전략 전체가 무의미. → 1.3장 5항 필수
3. **선택적 완화 레버(보류 가능)**: 해지 유저에게 "월 1편 추억 듣기" 무료 제공은 락인을 약화시키므로 기본 미적용. 분쟁률이 실제로 높을 때만 꺼내는 카드로 보관

### 2.3 광고 설계 — 정책 리스크 먼저
아이가 직접 조작하는 앱이므로 **Google Play 가족 정책 / Apple 아동 관련 심사**가 광고 설계를 지배한다. 대상 연령을 "혼합(어린이 포함)"으로 신고해야 하고, 그 경우 아동에게 노출되는 광고는 비개인화 + 가족 친화 등급만 허용되며 eCPM이 일반 앱 대비 크게 낮다. **광고는 보조 수익 + "광고 없애려면 구독" 압박 장치로 설계하고, 주 수익은 구독으로 잡는다.**

| 지면 | 형식 | 규칙 |
|---|---|---|
| 전 탭 상단 | 앵커드 적응형 배너 (AdMob) | 요청대로 상시 노출. 단 **재생 화면과 오디오 재생 중에는 미노출**(청취 UX 파괴 + 아이 오터치 유발 → 정책 리스크). 최대 광고 콘텐츠 등급 G, 비개인화 태그 필수 |
| 서재 리스트 3번째 슬롯 | 네이티브 광고 | 카드 디자인과 동일 프레임, "광고" 라벨 |
| 보상형 ① | "프리미엄 표지 테마 24시간 해금" | 부모가 시청하는 구조로 배치(재생 전 부모 확인 게이트) |
| 보상형 ② | "이야기 라디오 연속재생 30분 → 무제한(오늘)" | 무료 유저 타이머 제한과 짝을 이룸 |
| 금지 | 전면 광고, 앱 오프닝 광고, 오디오 광고 | 아동 UX·심사 리스크·수면 대체 수요 파괴. 넣지 않는다 |

### 2.4 마법의 책 수익성 판정: **광고 기반 = 적자 확정 → 구독 편입**
- 1회 생성 원가 ≈ ₩100~150 (2.2장)
- 가족 정책 준수 보상형 광고 1회 시청 수익 ≈ ₩5~30 (비개인화·G등급 인벤토리 기준 보수 추정)
- → **회당 ₩70~145 적자.** "광고 보면 사용" 모델은 기획에서 제외한다. 대신 구독 가치로 편입(월 10회)하면 원가 월 최대 ₩1,500으로 구독가 6,900원 안에서 충분히 흡수되고, 구독의 두 번째 셀링포인트가 된다. 기능 자체는 죽이지 않는다 — 죽는 건 광고 모델뿐이다.
- **v2.1 확정: 마법의 책은 우리집 목소리와 한 몸의 과금 단위다.** 산출물 잠금·복구 규칙(2.5장)을 그대로 적용하고, 비구독 체험 1회는 나레이터 목소리로만 생성해 회당 원가를 ₩100 이하로 묶는다. 월 10회 초과 수요는 소모성 회수권(2.6장)으로 흡수한다.

### 2.6 부가 수익 레버 (v2.1 신설) — "거부감 없는 추가 매출" 판정
주 수익(구독)을 해치지 않고, 가족 정책 리스크가 없으며, 선택적 지출이라 저항이 낮은 것만 채택한다.

| # | 레버 | 내용 | 원가/마진 | 판정 |
|---|---|---|---|---|
| R1 | 마법의 책 회수권 (소모성 IAP) | 월 10회 소진 시 +5회 ₩2,500 | 원가 ≈₩750, 마진 약 70% | **채택.** 구독자 한정 판매 — 비구독에 열면 구독 가치가 희석된다 |
| R2 | 스토리 팩 단권 판매 | 신규 팩(10편 단위)당 ₩3,900 일시불, 나레이터 음원 소장형. 구독자는 전 팩 자동 포함 | 사전 생성이라 한계원가 ≈0 | **채택.** 구독 저항층의 진입 상품. "구독엔 전 팩 포함" 명시로 카니발 방지 |

> **구현 현황 (2026-07-04 QA 검수 기준):** R1 서버 로직 구현됨·클라이언트 IAP 연동 미완 / R2·R4 미구현 /

### 2.7 기생성 콘텐츠(동화 서재·이야기 라디오) 외부 유통 전략 (v2.1 신설)
핵심 구분: **잠그는 자산은 부모 목소리, 무료 지면 자산은 나레이터 음원**이다. 나레이터 음원은 앱 안에서도 무료이므로 외부에 풀어도 락인 전략과 충돌하지 않는다 — 오히려 유입 퍼널이 된다.

| 채널 | 방식 | 수익 판정 |
|---|---|---|
| 유튜브(키즈) | Flow 표지 + 오디오를 영상으로 자동 변환, 편당 업로드. 설명란에 앱 링크 | 아동 콘텐츠 지정 시 개인화 광고 불가로 광고 수익은 소액. **주 목적은 유입 퍼널**, 수익은 부수입으로 판정 |
| 팟캐스트(스포티파이 등) | 팩당 앞 3~5편만 공개, "전편은 앱에서" 아웃트로 | 직접 수익 없음. 검색 유입 + 신뢰 자산 |
| 스마트 스피커 (클로바 등) | "동화 틀어줘" 스킬 | 후순위. 앱 안정화 후 검토 |

**선행 게이트(7장에 추가):** ElevenLabs 유료 플랜의 상업적 이용 라이선스와 Voice Design 제작 음성의 외부 플랫폼 배포 허용 여부를 출시 전 재확인할 것. 확정 나레이터 음성(4장 주석)의 권리 범위 확인 포함.

### 2.8 구독 전환 훅 & 어텐션 장치 (v2.1 신설)
전환의 공식: **"이 앱은 우리 아이를 안다"는 개인화 첫인상 → 정서 자산 축적의 시각화 → 잃기 싫은 순간에 구독 제안.**
모든 훅의 공통 제약: 부모 음성 외부 반출 금지(C2), 아동 광고 정책 준수, 한계원가 통제.

| # | 훅 | 내용 | 원가 | 판정 |
|---|---|---|---|---|
| H1 | **첫 3분 개인화** (온보딩) | 설치 직후 아이 이름 입력 → 나레이터가 "OO아, 만나서 반가워. 오늘은 어떤 이야기를 들어볼까?" 환영 오디오를 즉석 생성·재생 → 바로 추천 동화 1편 | 30자 내외, 편당 ≈₩5 | **채택.** 첫 실행 3분 내 "내 아이 이름을 불러주는 앱" 각인 — 최강의 전환 진입점 |
| H2 | 듣기 스탬프북 + 월간 리포트 | 아이용: 10편마다 스탬프·배지. 부모용: "이번 달 OO는 12편, 140분 들었어요" 월간 카드 | 0 | **채택.** 자산 축적 시각화 = 락인. 카드는 이미지만 공유 가능(음성 미포함) |

## 3. TTS 품질 진단 — "기괴한 느낌"의 원인 (50편 전수 분석 결과)

| # | 발견 | 근거 | 영향 |
|---|---|---|---|
| 1 | **50편이 50개의 서로 다른 목소리 프롬프트를 가짐** | 전 편의 elevenlabs_prompt가 전부 고유 ("oceanic voice", "doughy voice", "sandy voice"…) | 편마다 목소리 생성/변경 → 앱 전체 톤 불일치, 비용 낭비. 이 필드는 표준 TTS API에서 아무 효과도 없는 필드다(Voice Design용 문구) |
| 2 | **voiceId가 50편 중 1편(story_01)에만 존재** | 02~50 전부 누락 | 고정 보이스 없이 생성 → 매번 다른 화자 = "기괴함"의 1차 원인 |
| 3 | **물결표·모음 늘임 표기** | story_01: `~` 21회 ("하~얀", "너~~무", "안~녕~") | 한국어 TTS는 ~를 늘임으로 읽지 못하고 웅얼거림·글리치로 처리 → 억양 붕괴 |
| 4 | **말줄임표(…) 남용** | 49편 전 편에 1~7회, 특히 마지막 문장("스르르… 잠들었답니다") | ElevenLabs가 …에서 어미를 흐리며 끊음 → 사용자가 말한 "끝음 처리가 사람 같지 않은" 현상의 직접 원인 |
| 5 | **02~50 프롬프트가 전부 whisper/soothing 계열** | "hushed", "whispering", "deeply soothing" 일색 | 속삭임 지시 + 낮은 에너지 텍스트 = 생동감 없음. 신규 방향(유치원 선생님 톤)과 정반대 |
| 6 | 문체 이원화 | story_01만 의성어·감탄사형, 02~50은 서술형 | 톤 불일치. 단 02~50 텍스트 품질 자체는 준수 — 전면 재작성 불요, 낭독 연출로 해결 |

### 3.1 해결책 (동봉 stories_v2.json에 반영 완료)
1. **보이스 통일**: 스토리별 프롬프트 폐기 → 전역 voice_profiles 3종(포근한 선생님/신나는 선생님/우리집 목소리). 나레이터 보이스는 ElevenLabs Voice Design으로 1회 제작 후 voice_id 고정(제작 프롬프트 6.3장)
2. **tts_text 필드 신설**: 화면용 content와 낭독용 텍스트 분리. ~ 전면 제거, … → `<break>` 태그 변환(문중 0.6s / 문말 1.0s), 문단 사이 1.2s 휴지, 느낌표 연타 정리, 문장부호 보정 — 50편 전부 적용 완료
3. **보이스 설정 기준값**: stability 0.45~0.55(너무 높으면 밋밋, 낮으면 불안정), style 0.30~0.45(생동감 구간), speed 0.93. 부모 목소리는 stability 0.55/style 0.15(클론은 안정 우선)
4. **끝음 마감**: 모든 트랙 후처리로 끝 무음 트리밍 + 1.5초 페이드아웃 + -16 LUFS 정규화 → "뚝 끊기는" 체감 제거
5. **표현력 상향 옵션**: eleven_v3 모델은 `[giggles]` `[whispers]` `[excited]` 오디오 태그로 유치원 선생님 연기가 가능. 단 알파 성격상 안정성 편차가 있으므로 **기본은 v2, 나레이터 라이브러리(사전 생성분)만 v3로 A/B** 후 승자 채택
6. **자동 QA**: 생성 오디오를 OpenAI Whisper로 받아써서 원문과 대조 → 오독·누락 5% 초과 시 자동 재생성 (6.6장)

### 6.1 Antigravity 마스터 프롬프트 (프로젝트 첫 메시지)
```

데이터 원본은 assets/data/stories_v2.json 단일 파일이고, 스토리·목소리 설정을 절대 하드코딩하지 않는다.
```
